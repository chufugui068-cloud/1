# 下一步执行清单（从0到可跑通）

> 目标：用最短路径做出第一个可演示版本（MVP）

## 第0步：先把开发环境跑起来（今天就做）
1. 注册并创建微信小程序（若已创建可跳过）。
2. 安装微信开发者工具，创建本地项目目录。
3. 在微信云开发里创建环境（建议 `test` 与 `prod` 两套）。
4. 开启云函数与云数据库权限。

**验收标准**：能在开发者工具里成功调用一个“hello world”云函数。

## 第1步：先做“今日比赛列表”页面（第1-2天）
1. 建立页面：`pages/matches/index`。
2. 接口先用假数据（mock），字段与 `GET /api/matches/today` 保持一致。
3. 页面展示：联赛、时间、主客队、更新时间。
4. 支持筛选：联赛 + 开赛时间。

**验收标准**：列表可加载、可筛选、点击可进入详情。

## 第2步：做“比赛预测详情”页面（第3-4天）
1. 建立页面：`pages/predict/detail`。
2. 接入 `GET /api/predict/detail`（先mock后真实）。
3. 展示内容：
   - 胜平负 / 让球概率
   - 总进球区间
   - TOP3比分
   - 推荐理由
4. 加入免责声明组件（底部固定）。

**验收标准**：单场可完整显示预测信息。

## 第3步：先实现最小模型服务（第5-6天）
1. 先不追求复杂，先把链路跑通：
   - ELO：基础更新公式
   - BT：由评分差映射概率
   - MC：5000次比分采样
2. 输出统一结构，供云函数直接返回前端。
3. 固定一个 `model_version`，便于追踪命中率。

**验收标准**：给任意一场比赛可返回完整概率与TOP3比分。

## 第4步：历史记录与赛后回写（第7天）
1. 每次预测写入 `predictions`。
2. 赛后写入 `prediction_results`。
3. 做一个历史页：显示命中与未命中。

**验收标准**：至少有“预测->赛后结算->历史展示”的完整闭环。

## 第5步：VIP先做“权限开关”，后接支付（第2周）
1. 免费用户仅看基础信息。
2. VIP可看TOP3、详细理由、历史复盘。
3. 暂无商户号时，先完成订单表和状态流转。

**验收标准**：同一页面可根据VIP状态展示不同内容。

## 第6步：数据抓取上线前检查（必须做）
1. 抓取频率限流（例如 5~10 分钟一次）。
2. 增加失败重试与缓存。
3. 主源失败时切换备用源。
4. 留存抓取日志（便于排障）。

**验收标准**：连续运行24小时抓取稳定。
